# 自动驾驶数据采集平台

可在 Windows 上一键运行的自动驾驶数据采集管理平台。

## 快速开始

双击 `run.bat`，会自动安装依赖、启动服务并打开浏览器（http://127.0.0.1:8080）。

或手动运行：

```bat
pip install -r backend\requirements.txt
python backend\main.py
```

## 功能

| 功能 | 说明 |
|------|------|
| 🗺️ 地图 | Leaflet + 高德瓦片（标准图/卫星图切换），内置 WGS-84↔GCJ-02 坐标纠偏 |
| 📍 打点 | 点击地图添加采集点，支持类型/天气/光照/路况场景标注 |
| 🛣️ 路径 | 地图上连续点击绘制采集路线，自动计算里程 |
| 📤 任务下发 | 创建任务→绑定路线→下发给车辆（可自动分配空闲车辆） |
| 🔥 热力图 | 采集轨迹密度热力覆盖，一键开关 |
| 🚗 车辆监控 | 多车实时位置/速度/电量，内置移动模拟器演示 |
| 🎬 轨迹回放 | 点击车辆弹窗中"轨迹回放"按钮回放历史轨迹 |
| 🚧 电子围栏 | 绘制多边形采集区域 |
| ⚠️ 告警 | 任务完成/低电量/异常告警，未读红点提示 |
| 📊 仪表盘 | 车辆/点位/里程/任务完成率/覆盖网格统计 |
| 📎 附件上传 | 点击采集点弹窗中"📎 附件"，可上传/下载/删除文件（照片、传感器日志等，单文件限 100MB，存于 `backend/uploads/`） |
| 💾 数据包管理 | 完整采集数据管理流水线：数据包创建 → 多文件上传（进度条 + SHA-256 校验 + 去重 + 按扩展名自动分类相机/激光雷达/定位/总线/日志）→ 自动质检（完整性/传感器完备性/空文件/丢帧率/时间同步）→ 标签管理 → 状态/标签/关键字检索 → 清单导出（manifest.json）/整包 ZIP 下载 → 归档/恢复生命周期；任务采集完成后车端自动注册数据包 |
| 📁 导出 | 点位 CSV、点位+路径 GeoJSON |

## 技术栈

- 后端：Python FastAPI + SQLite（数据文件 `backend/platform.db`，删除即可重置示例数据）
- 前端：原生 JS + Leaflet 1.9 + leaflet.heat
- 车辆模拟器：后台线程沿任务路线移动车辆、记录轨迹、生成告警

## 项目结构

```
adas-data-platform/
├── backend/          # FastAPI 后端（API + 静态托管）
│   ├── main.py       # 路由入口
│   ├── database.py   # SQLite 表结构
│   ├── models.py     # 请求模型
│   ├── datasets.py   # 采集数据包管理（上传/质检/标签/检索/导出/归档）
│   ├── seed.py       # 示例数据（上海）
│   └── simulator.py  # 车辆移动模拟
├── frontend/         # 前端页面
└── run.bat           # 一键启动
```

## 打包发布

- **绿色 EXE**：`dist\ADAS数据采集平台.exe`（PyInstaller 单文件，目标机免装 Python，双击即启动并自动打开浏览器；数据库/上传文件生成在 exe 同目录）
- 重新打包：`python -m PyInstaller --onefile --name "ADAS数据采集平台" --add-data "<项目绝对路径>\frontend;frontend" backend\main.py`

## 说明

- 高德瓦片使用 GCJ-02 坐标系，平台内部数据统一存 WGS-84，前端自动纠偏。
- 商用部署请遵守高德地图服务条款并申请正式 Key/许可。
