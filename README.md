## 🚗 模块一：ADAS 数据采集平台（含完整源码）

一站式采集管理平台，覆盖 **任务规划 → 出车检查 → 车端采集 → 数据回传 → 质检 → 治理消费** 全流程。

**地图与采集作业**
- 高德地图底座（标准/卫星图切换），数据存 WGS-84、前端自动 GCJ-02 纠偏
- 采集打点（类型 + 天气/光照/路况场景标注 + 现场照片附件）、路线绘制（自动算里程）、电子围栏
- 轨迹热力图一键开关；覆盖缺口网格分析，高亮采集不足区域指导补采

**任务与车队管理**
- 任务创建 → 绑定车辆/路线/采集员/传感器配置 → 下发 → 进度实时跟踪，四级优先级
- 出车检查单：下发前 6 项 SOP 强制确认（标定/镜头/自检/天线/存储/时同），未确认拒绝下发
- 事件触发规则（如 `AEB:10:30` 截取触发前 10s 后 30s）；Campaign 一套配置多车批量生成任务
- 采集员管理（指派到人、状态与工作量统计）；传感器配置模板；车辆实时监控与轨迹回放

**数据包管理**
- 生命周期状态机：回传中 → 已上传 → 质检通过/未通过 → 归档/恢复
- 分片断点续传上传：2MB 分片、中断续传自动跳过已收分片、SHA-256 校验去重、按传感器自动分类
- 可配置质检 8 项（完整性/丢帧率/时间同步/相机曝光/点云密度/GNSS 失锁等），不合格一键生成补采任务
- 合规脱敏标记、优先级回传调度、标签/状态/事件多维检索、数据血缘（下游消费记录）、保留策略自动归档、manifest/ZIP 导出

**运营与治理**
- 概览仪表盘（8 项 KPI + 14 天趋势 + 最近告警）；里程/数据量/合格率/任务数趋势报表
- 存储水位监控、操作审计日志、告警中心

**技术实现**
- 后端 Python FastAPI + SQLite，内置车辆移动/任务进度/数据回传模拟器，无需真车即可完整演示
- 前端原生 JS + Leaflet，零框架依赖

```bash
cd adas-data-platform && run.bat   # → http://127.0.0.1:8080
```

## 🌐 模块二：数据闭环平台（`数据平台.html`）

数据闭环的中枢 —— 采集数据切片、场景生成与标签挖掘的场景管理平台。

**场景资产管理**
- WorldSim / LogSim 双类型场景数据统一管理，列表与缩略图双视图切换
- 场景元信息完整展示：名称、来源、时长、创建时间、标签集

**检索与挖掘**
- 标签组合检索：多标签叠加过滤，精确圈定目标场景集
- 时间区间筛选 + 关键字搜索，三重条件组合定位
- 支撑corner case 挖掘：从海量路采数据中快速筛出高价值场景

**场景操作**
- 场景详情查看、可视化轨迹预览
- 手动打标：为场景补充/修正业务标签
- 标准格式下载：xosc（OpenSCENARIO）/ xodr（OpenDRIVE）场景文件导出

**仿真衔接**
- 场景一键导出至仿真平台生成测试用例，打通"数据 → 用例"链路
- 场景与用例双向关联，导出记录可追溯

## 🧪 模块三：仿真测试平台（`仿真平台.html`）

数据闭环的验证端 —— 面向自动驾驶算法（AEB/导航/避障等）仿真验证的测试平台。

**用例管理**
- 卡片式用例视图：场景缩略图、用例名称、创建时间一目了然
- 用例全操作闭环：新建、运行、编辑、删除

**搜索与筛选**
- 关键字搜索 + 多条件组合筛选，海量用例中快速定位
- 按运行状态过滤，聚焦待处理用例

**运行与评价**
- 状态标签颜色区分：待运行 / 运行中 / 通过 / 失败
- 用例批量运行与结果统计，覆盖率与通过率一览
- 测试结果沉淀：失败用例对应的问题场景反哺场景挖掘与补采，形成闭环

> 两个静态页面均为纯 HTML + CSS + 原生 JS 单文件，无外部依赖，双击即可打开（建议 1920×1080 及以上分辨率）。
