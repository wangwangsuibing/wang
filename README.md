## 🔄 全流程数据闭环

```
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│  ① 数据采集   │ ───▶ │  ② 数据管理   │ ───▶ │  ③ 仿真测试   │
│ ADAS采集平台  │      │  数据闭环平台  │      │  仿真测试平台  │
└──────────────┘      └──────────────┘      └──────────────┘
  任务·路采·质检         场景挖掘/打标           用例回归/评价
        ▲                                          │
        └────────── 问题场景反哺采集与算法迭代 ◀────┘
```

- **① 数据采集**：任务规划下发、车端多传感器采集、数据回传与质检，产出合格数据包；
- **② 数据管理**：原始数据切片、Log2World 场景生成、标签化管理与检索挖掘；
- **③ 仿真测试**：场景导出生成用例，对算法（AEB/导航/避障等）回归测试；
- **闭环反哺**：测试暴露的问题场景反向指导补采与场景挖掘，驱动算法持续迭代。

# 🚗 核心模块：ADAS 数据采集平台（含完整源码）

一站式采集管理平台，覆盖 **任务规划 → 出车检查 → 车端采集 → 数据回传 → 质检 → 治理消费** 全流程。

## 一、地图与采集作业

| 功能 | 详细说明 |
|------|----------|
| 🗺️ 地图底座 | Leaflet + 高德瓦片（标准图/卫星图切换）；数据统一存 WGS-84，前端自动做 GCJ-02 纠偏，坐标零漂移 |
| 📍 采集打点 | 点击地图添加采集点，支持点位类型（兴趣点/事件点/障碍物/起终点）+ 场景标注（天气/光照/路况三维度），可关联任务、按任务筛选展示 |
| 🛣️ 路线绘制 | 地图连续点击绘制采集路线，自动计算里程，作为任务执行路径 |
| 🚧 电子围栏 | 绘制多边形采集区域，圈定作业范围 |
| 🔥 轨迹热力图 | 采集轨迹密度热力覆盖，一键开关，直观呈现采集浓度分布 |
| 🕳 覆盖缺口分析 | 轨迹按地理网格聚合，自动识别并在地图高亮**采集不足的网格**，指导补采排班 |
| 📎 点位附件 | 采集点可上传/下载/删除现场照片、传感器日志等文件（单文件限 100MB） |

## 二、任务与车队管理

| 功能 | 详细说明 |
|------|----------|
| 📤 任务管理 | 任务创建 → 绑定车辆（或自动分配空闲车）/路线/采集员/传感器配置 → 下发 → 执行进度实时跟踪（进度条）→ 完成/终止；四级优先级（低/中/高/紧急） |
| 📋 出车检查单 | 下发前强制 6 项 SOP 确认（传感器标定/镜头清洁/设备自检/GNSS 天线/存储空间/时间同步），**未确认后端拒绝下发**，从流程上杜绝废数据 |
| ⚡ 事件触发规则 | 任务可配置事件触发采集规则，如 `AEB:10:30` 表示 AEB 触发时截取前 10s 后 30s 数据，支持多条规则（AEB/切入/手动打标） |
| 📦 Campaign 采集活动 | 一套配置（路线 + 传感器方案）勾选多辆车，**一键批量生成任务**，适配专项采集（如"Q3 雨天专项"） |
| 👤 采集员管理 | 采集员增删、状态跟踪（空闲/任务中，任务完成自动释放）、完成单数统计，任务指派到人 |
| 🎛 传感器配置方案 | 预置"标准 6V1L 套件""高精采集套件"等传感器配置模板，任务/活动直接引用，保证采集配置一致性 |
| 🚗 车辆实时监控 | 多车实时位置/速度/电量/状态（空闲/采集中/离线），内置移动模拟器沿任务路线行驶演示 |
| 🎬 轨迹回放 | 点击地图车辆 → 一键回放历史轨迹动画 |

## 三、数据包管理（数据回传与质检）

| 功能 | 详细说明 |
|------|----------|
| 💾 数据包生命周期 | `回传中 → 已上传 → 质检通过/未通过 → 归档/恢复` 完整状态机；任务完成后车端自动注册数据包 |
| 🔄 分片断点续传 | 大文件按 2MB 分片上传：`init`（建会话）→ 逐片 `PUT`（中断后重传**自动跳过已收分片**）→ `complete`（合并 + SHA-256 完整性校验 + 哈希去重）；文件按扩展名自动分类为相机/激光雷达/毫米波/定位/总线/日志 |
| 📶 回传调度 | 数据包按优先级排队回传（高优先级先传），列表实时显示回传进度百分比 |
| 🔍 可配置质检 | 8 项自动检查：文件完整性、传感器完备性、空文件、丢帧率、时间同步误差、**相机曝光异常、激光点云密度、GNSS 失锁**；丢帧/时同阈值与及格分数**在线可配**，质检报告逐项展示 |
| 🔁 一键补采 | 质检不合格数据包一键生成**高优先级补采任务**（自动复用原任务的路线/车辆/传感器配置） |
| 🔒 合规脱敏 | 数据包人脸/车牌脱敏状态管理（待脱敏/已脱敏/无需脱敏），含视觉数据强制标记 |
| 🏷 标签与检索 | 标签管理 + 多维组合检索：关键字/状态/标签/事件类型（AEB、切入、手动打标）四重筛选 |
| 🧬 数据血缘 | 记录数据包被下游消费的完整历史（标注平台/训练平台/仿真平台/外部交付 + 备注），支撑合规审计与影响分析 |
| 🗄 保留策略 | 可配置"N 天未被消费自动归档"，在线修改并可立即执行，控制存储成本 |
| 📄 导出交付 | manifest.json 清单导出（含全部文件哈希）+ 整包 ZIP 下载；点位 CSV / GeoJSON 导出 |

## 四、运营与治理

| 功能 | 详细说明 |
|------|----------|
| 📊 概览仪表盘 | 8 项 KPI 卡片（车辆/任务/数据包/数据量/点位/路线/轨迹点/覆盖网格）+ 近 14 天趋势图 + 最近告警，一屏掌握运营全貌 |
| 📈 趋势报表 | 近 14 天**采集里程/数据量/质检合格率/完成任务数**四组趋势柱状图（纯 CSS 实现，零图表库依赖） |
| 💽 存储水位 | 磁盘用量实时监控 + 可视化水位条 + 告警线超限提示 |
| 🧾 审计日志 | 任务下发、数据删除、质检规则修改、保留策略变更等关键操作全程留痕（操作/对象/详情/时间） |
| ⚠️ 告警中心 | 任务完成/低电量/异常告警，导航栏未读红点，进入自动已读 |

## 五、界面设计（对标业界规范）

- **左侧深色导航**（208px，可折叠至 52px 图标栏）组织 7 大模块：实时监控 / 概览仪表盘 / 任务管理 / 数据管理 / 车辆与人员 / 报表分析 / 告警中心
- **实时监控页**：地图全屏 + 右上角浮动 KPI 卡 + 可折叠在线车辆面板（Samsara 风格）
- **任务/数据管理**：标准表格列表页 —— 筛选栏 + 表格（hover 高亮/操作列）+ 空状态引导
- **创建表单与详情**：全部右侧 Drawer 滑出（Ant Design 规范），详情 Drawer 内分"文件/质检/血缘"Tab；危险操作保留居中确认弹窗
- **统一色板**：Ant Design 色板（主色 `#1677ff`），8px 圆角卡片，深色导航 `#001529`

## 六、技术栈与运行

- **后端**：Python + FastAPI + SQLite（WAL 模式）；内置车辆移动/任务进度/数据回传三重模拟器，无需真车即可完整演示
- **前端**：原生 HTML/CSS/JavaScript（**零框架依赖**）+ Leaflet 1.9 + leaflet.heat
- **数据**：内置上海示例种子数据（车辆/路线/任务/采集员/传感器配置）；删除 `backend/platform.db` 即重置

```bash
git clone https://github.com/wangwangsuibing/wang.git
cd wang/adas-data-platform
run.bat        # 自动安装依赖并启动 → http://127.0.0.1:8080
```

```
adas-data-platform/
├── backend/
│   ├── main.py       # 路由入口（任务/车辆/点位/围栏/报表/审计等 API）
```

# 🌐 模块二：数据闭环平台（`数据平台.html`）

数据闭环的中枢 —— 采集数据切片 → 场景生成 → 标签挖掘的场景管理平台 Demo：

- **场景资产管理**：WorldSim / LogSim 场景数据列表，列表与缩略图双视图切换
- **智能检索**：标签组合检索 + 时间区间筛选 + 关键字搜索，支撑场景挖掘
- **数据操作**：场景详情、可视化轨迹预览、手动打标、xosc/xodr 场景文件下载
- **仿真衔接**：场景一键导出至仿真平台生成测试用例
- **暗色主题**：企业级 B 端暗色视觉，适配长时间数据分析

# 🧪 模块三：仿真测试平台（`仿真平台.html`）

数据闭环的验证端 —— 面向自动驾驶算法（AEB/导航/避障等）仿真验证的测试平台前端 Demo：

- **用例管理**：卡片式展示仿真测试用例（场景缩略图/名称/创建时间）
- **搜索筛选**：多条件组合筛选，快速定位目标用例
- **状态标识**：颜色标签区分用例状态（待运行/运行中/通过/失败）
- **测试闭环**：新建/运行/编辑/删除操作闭环，测试结果反哺场景挖掘与算法迭代

> 两个静态页面均为纯 HTML + CSS + 原生 JS 单文件，无外部依赖，双击即可用 Chrome/Edge 打开（建议 1920×1080 及以上分辨率）。
